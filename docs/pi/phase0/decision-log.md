## Required Wording To Carry Forward

Security boundary:

> T3 forwards prompts and selected environment/cwd into Pi. Pi-side tools, slash commands, prompt templates, skills, extensions, model routing, auth, and upstream-provider behavior remain governed by Pi configuration. T3 extension UI bridging is interactivity, not a complete T3 approval sandbox.

Upstream provider compliance:

> Pi may route prompts to upstream providers based on Pi-side configuration. T3 does not validate upstream provider entitlement, billing, official-harness requirements, or terms of service. Users are responsible for configuring Pi and any upstream providers they use through Pi.

Raw logging privacy:

> Persisted Pi raw logs must be redacted or debug-gated. Pi RPC events can include prompts, tool arguments, file paths, extension payloads, and provider metadata. Fixture files must anonymize secrets, local paths, response ids, encrypted reasoning signatures, and PII.

Fresh runtime UX:

> Pi runtime restarted; previous Pi agent context was not resumed. T3 conversation history is still visible, but Pi is starting from a fresh runtime context.
